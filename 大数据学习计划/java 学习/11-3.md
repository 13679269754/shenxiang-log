## 修饰符

**访问修饰符**
public/private/protected

| 修饰符         | 同一个类中 | 同一个包中 | 不同包中的子类 | 不同包中的非子类 |
| ----------- | ----- | ----- | ------- | -------- |
| `private`   | ✅     | ❌     | ❌       | ❌        |
| 默认（无修饰）     | ✅     | ✅     | ❌       | ❌        |
| `protected` | ✅     | ✅     | ✅       | ❌        |
| `public`    | ✅     | ✅     | ✅       | ✅        |

setter 与getter方法提供属性
- 将字段用 `private` 修饰，从而更好地将信息进行封装和隐藏。
- 用 `setXXXX` 和 `getXXXX` 方法对类的属性进行存取，分别称为 **setter** 与 **getter**。
- 这种方法有以下优点：
  1. 属性被 `private` 修饰后更好地封装和隐藏，外部类不能随意干扰。
  2. 提供方法来存取对象的属性，在方法中可以对给出的参数的合法性进行检查。
  3. 方法可以用来给出计算后的值。
  4. 方法可以完成其他的操作（如清理资源、设定状态等）。
  5. 只提供 `get` 方法，不提供 `set` 方法的属性，可以保证属性是只读的。

```
class Person2 {
    private int age;

    public void setAge(int age) {
        if (age > 0 && age < 200) {
            this.age = age;
        }
    }

    public int getAge() {
        return age;
    }
}

```


**其他修饰符**
abstract/static/final
以上修饰符可以并存

| 修饰符        | 基本含义          | 可修饰类 | 可修饰成员 | 可修饰局部变量 |
| ---------- | ------------- | ---- | ----- | ------- |
| `static`   | 静态的、非实例的、属于类的 | 内部类  | ✅     | ✅       |
| `final`    | 最终的、不可改变的     | ✅    | ✅     | ✅       |
| `abstract` | 抽象的、不可实例化的    | ✅    | ✅     | ❌       |
tips:
statis
- static 方法只能使用static域中的属性，调用static方法。
- static 不能访问实例变量，不能使用this和super。
- 调用时直接使用类名。

final
- final类不能被继承
- final方法不能被子类覆盖（不能被影藏）

abstract
- 抽象类不能被实例化
- 抽象方法不需要具体实现；
- 抽象方法所在的类必须是抽象类，抽象类可以包含抽象方法；
- 抽象方法在子类中必须被实现，否则子类也是抽象类。


## 接口（interface）

- 接口都是public abstract 的 
- 接口命名一般为able和ible结尾
- 接口可以多继承，但是不需要存在类与接口的继承关系
- `[public]  interface interfaceName [extends listOfSuperInterface]` public 修饰后说明任意类可访问，缺省情况下只有与该接口定义在一个包中的类才可以访问。
- 一个接口可以有多个父接口，一个类只能有一个父类。

**接口实现**
implements 

- 类体中必须实现所有接口中定义的方法。
- 一个类可以实现多个接口，逗号`,`分割。
- 接口中的常量是 **publlic,static,final**的
- java8 中可以实现static方法，可以实现default方法（默认方法，可以提供一个默认的方法实现）
  
**变量传递**
- java 是值传递，
	- 当传递基本类型时，传递的是值的副本；
	- 当传递对象（包括数组）时，传递的是**引用的副本**。
